# рҹ“ҡ Todaii English вҖ” Backend

> A multi-module Spring Boot backend powering an AI-driven English learning platform with TOEIC exam preparation, personalized study plans, streaming chatbot, and smart dictionary.

The system is designed as two independently deployable Spring Boot applications вҖ” **User API** and **Admin API** вҖ” sharing a common core, infrastructure, and domain layer via Maven multi-module architecture.

---

## вңЁ Features

### рҹӨ– AI-Powered Learning Engine

- **Personalized Study Plans** вҖ” Scheduled job (`@Scheduled`, every 2 days at 07:00) analyzes each user's TOEIC weakness, study streak, daily study time, and target score to generate AI-tailored study plans via prompt templates.
- **Dual AI Provider with Fallback** вҖ” `AiFallbackService` tries OpenAI first, automatically falls back to Gemini if the primary fails, and logs every token usage to `usage_statistics` for cost auditing.
- **Custom Spring AI Advisors** вҖ” `TokenUsageCallAdvisor` (call) and `TokenUsageStreamAdvisor` (stream) intercept every AI request to audit prompt/completion tokens.
- **Streaming Chatbot** вҖ” Real-time conversational AI using `Flux<String>` streaming with configurable timeouts, chat memory persisted to MySQL via JDBC (`MessageWindowChatMemory` with `JdbcChatMemoryRepository`), and provider selection (OpenAI / Gemini).

### рҹ“қ TOEIC Exam System

- **Full Session Management** вҖ” Start, save-in-progress, submit workflow with optimized batch answer processing (single-query question lookup, upsert strategy).
- **Official Score Conversion** вҖ” Non-linear score conversion tables mapping raw correct answers to official TOEIC Listening/Reading scores (5вҖ“495).
- **Performance Analytics** вҖ” Per-part accuracy breakdown (Parts 1вҖ“7) for radar charts, weakest-part identification, score prediction based on last 3 completed full tests with trend analysis (IMPROVING / STABLE / DECLINING).
- **CEFR Level Mapping** вҖ” Automatic mapping of predicted TOEIC scores to CEFR levels (A1вҖ“C2).
- **Smart Recommendations** вҖ” Content suggestions (articles, videos, practice tests) personalized by CEFR level and weakest TOEIC parts.

### рҹ“– Smart Dictionary

- **Cache-Aside Strategy** вҖ” Three-tier lookup: Redis cache вҶ’ MySQL database вҶ’ External API call (Todaii Dictionary API), with automatic cache population and TTL refresh on hit.
- **Trending Words** вҖ” Redis Sorted Set tracking search frequency, exposing top-N most searched words.
- **Free Dictionary API** integration as a secondary lookup source.
- **AI-Powered Suggestions** вҖ” Context-aware word suggestions generated by the AI fallback pipeline.

### рҹ”” Engagement & Notifications

- **Study Streak Tracking** вҖ” Daily ping-based study time accumulation, streak calculation, and longest-streak tracking.
- **Streak Risk Alerts** вҖ” Daily scheduled job (19:00) warns users about to lose their streak via in-app notifications and email (max 3 reminders per cycle).
- **Exam Countdown** вҖ” Scheduled job (08:00 daily) sends milestone reminders (7, 5, 3, 1 days before exam date).
- **Email Delivery** вҖ” Async SMTP integration for study plan delivery, streak warnings, and exam countdowns.

### рҹ“° Content Management

- **Articles** вҖ” CEFR-tagged bilingual articles with paragraph-level structure.
- **Videos** вҖ” YouTube-synced video content with SRT subtitle parsing (Submerge) and lyric-line entities.
- **Vocabulary Decks** вҖ” Grouped vocabulary decks with performance monitoring via AOP (`VocabPerformanceMonitorAspect`).
- **Topics** вҖ” Categorized content organization.
- **Comments** вҖ” User-generated comments with reporting system.
- **Notebook** вҖ” Personal notebook for saving articles, videos, and vocabulary.

### рҹ”җ Security & Auth

- **JWT Authentication** вҖ” Custom `JwtTokenFilter` + `JwtUtility` with configurable issuer and secret.
- **Role-Based Access Control** вҖ” Separate security configurations for User API and Admin API with `AdminRole` entity.
- **Refresh Token Rotation** вҖ” Dedicated `UserRefreshToken` / `AdminRefreshToken` entities.

### вҳҒпёҸ Integrations

- **Google Cloud Translate** вҖ” VietnameseвҖ“English translation via service account credentials.
- **YouTube Data API v3** вҖ” Video metadata retrieval.
- **NewsAPI** вҖ” English news article sourcing.
- **Cloudinary** вҖ” Media file upload and management.
- **OpenAI TTS** вҖ” Text-to-speech audio generation.

**Module dependency graph:** `user-api` / `admin-api` вҶ’ `infra` вҶ’ `core` вҶ’ `shared`

The architecture follows a **ports-and-adapters** (hexagonal) pattern: `core` defines port interfaces (e.g., `DictionaryPort`, `CloudinaryPort`, `TtsPort`), while `infra` provides concrete implementations (API clients, Redis clients, security filters).

## рҹҡҖ Getting Started

### Prerequisites

- **Java 21** (Eclipse Temurin recommended)
- **Maven 3.9+**
- **MySQL 8+**
- **Redis** (or [Upstash](https://upstash.com/) for managed Redis)
- **Docker** (optional, for containerized deployment)

### Clone

```bash
git clone https://github.com/vantu2004/todaii-english-BE.git
cd todaii-english-BE
```

### Environment Configuration

Create `.env` files in `apps/user-api/` and `apps/admin-api/` based on the following template:

```env

# в”Җв”Җ Database в”Җв”Җ

DB_DRIVER=com.mysql.cj.jdbc.Driver
DB_USERNAME=root
DB_PASSWORD=your_db_password

JPA_DDL_AUTO=update
JPA_SHOW_SQL=false

# в”Җв”Җ Email (SMTP) в”Җв”Җ

# в”Җв”Җ AI Providers в”Җв”Җ
```

### Build & Run

**Using Maven (local development):**

```bash
# Build all modules
mvn clean install -DskipTests

# Run User API
mvn spring-boot:run -pl apps/user-api

# Run Admin API (in a separate terminal)
mvn spring-boot:run -pl apps/admin-api
```

**Using Docker:**

```bash
# Build & run User API
docker build -f apps/user-api/Dockerfile -t todaii-user-api .
docker run -p 8081:8081 --env-file apps/user-api/.env todaii-user-api

# Build & run Admin API
docker build -f apps/admin-api/Dockerfile -t todaii-admin-api .
docker run -p 8082:8082 --env-file apps/admin-api/.env todaii-admin-api
```
